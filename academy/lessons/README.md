# Lessons

Lesson content = the **source of truth** for the course, one markdown file per
lesson, named `MM-LL-slug.md` where `MM` = module number and `LL` = lesson
number within the module (e.g. `05-02-beautiful-deleveraging.md`).

Each lesson follows the **lesson anatomy** in [../CURRICULUM.md](../CURRICULUM.md)
(Hook → Intuition → Mechanism → In the dashboard → Try it → Check → New words)
and teaches Ray's concepts in original prose per the copyright rule in
[../ray_sources.md](../ray_sources.md) — never reproduce book text.

The `/academy` route (`dashboard/academy.py`, to be built) renders these on
live data. Markdown stays the canonical form so lessons are reviewable,
diffable, and exportable to a site or PDF later.
