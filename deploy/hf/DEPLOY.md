# Deploying the public dashboard to Hugging Face Spaces (free)

This puts the Economic Machine Dashboard on a public, always-on URL like
`https://huggingface.co/spaces/<you>/economic-machine-dashboard` for **$0**,
**no credit card**. It runs in read-only `PUBLIC_MODE` — safe for many
simultaneous viewers.

The Space needs only **three files**:

| File | What it is |
| :--- | :--- |
| `Dockerfile` | in this folder — clones the app code from GitHub at build time |
| `README.md` | in this folder — the Space's metadata (sets the port to 7860) |
| `emd_data.tar.gz` | the data bundle (compacted DuckDB + cache) — see step 4 |

The app code is **not** uploaded — the Dockerfile clones it from the public
GitHub repo. Only the data lives in the Space.

---

## One-time setup (~5 minutes)

### 1. Make a Hugging Face account
Sign up at https://huggingface.co/join (free, no card). Verify your email.

### 2. Create a new Space
- Go to https://huggingface.co/new-space
- **Owner:** you · **Space name:** `economic-machine-dashboard`
- **License:** MIT
- **SDK:** choose **Docker** → **Blank**
- **Hardware:** *CPU basic* (free) · **Visibility:** *Public*
- Click **Create Space**.

### 3. Upload `Dockerfile` and `README.md`
On the new Space page → **Files** tab → **Add file → Upload files**.
Drag in **both** `deploy/hf/Dockerfile` and `deploy/hf/README.md` from this
repo. Commit.
> The Space's `README.md` **replaces** the auto-generated one — that's correct;
> its front-matter is what tells HF to serve on port 7860.

### 4. Upload the data bundle `emd_data.tar.gz`
This is generated by the operator (see "Refreshing the data" below). Upload
it the same way (Add file → Upload files). Hugging Face stores large files
automatically — no git-lfs knowledge needed. Commit.

### 5. Watch it build
The Space switches to **Building** (2–4 min the first time). When it flips to
**Running**, click **App** (or the Space's public URL). You're live.

### 6. (Optional) Turn on the traffic page
`/traffic` is hidden by default in public mode. To see it:
- Space → **Settings** → **Variables and secrets** → **New secret**
- Name `TRAFFIC_KEY`, value = any password you pick (e.g. `emd-2026`)
- The Space restarts. View your metrics at
  `https://<your-space-url>/traffic?key=emd-2026`
- (Traffic counts reset whenever the Space rebuilds/restarts — it's a live
  counter, not a permanent log.)

## Refreshing the data (operator, whenever you want it current)

The public Space shows a **frozen snapshot**. To publish fresh numbers, on the
machine that holds the live database:

```bash
# from the project root — compacts the 2GB working DB down to ~67MB and
# bundles it with the cache into emd_data.tar.gz
python scripts/build_public_bundle.py
```

Then re-upload the new `emd_data.tar.gz` to the Space (step 4). The Space
rebuilds automatically and serves the fresh data. Nothing else changes.

---

## Notes & limits (free tier)

- **Always-on** while there's occasional traffic; HF may pause a Space after
  ~48h of *zero* visits — the next visitor wakes it (a few seconds).
- **16 GB RAM / 2 vCPU** free — comfortably more than this app needs.
- **No database writes happen in the Space** — it's a pure reader, so multiple
  viewers never conflict.
- **Custom domain / no HF chrome:** if you later want `dashboard.yoursite.com`
  instead of the `hf.space` URL, that's a paid HF feature or a reverse-proxy;
  the free `hf.space` URL is perfectly shareable for a class.
